PROJECT: L-CDEA
MODULE: ingestion/chunking/

PURPOSE:
Fix dictionary-mode chunking so each valid dictionary/glossary entry becomes its own independently traceable chunk.

Current behavior:
  mini_dictionary.txt contains 4 definitions.
  ingest_document(..., mode="dictionary") reports:
    chunks_processed = 1
    definitions = 4
    nodes_added = 4

This means the whole file is treated as one chunk, while extractors split multiple definitions internally.

This works functionally, but weakens provenance because multiple definitions share one chunk context.

Desired behavior:
  In dictionary mode, each valid definition line should become its own chunk.

Example input:
  Velocity is displacement over time.
  Force is mass times acceleration.
  Acceleration is the rate of change of velocity.
  Mass is the amount of matter in an object.

Expected:
  chunks_processed = 4
  definitions = 4
  nodes_added = 4

Each definition should have its own stable chunk_id.

# DICTIONARY MODE CHUNKING RULE

If mode == "dictionary":

1. Split document by lines.
2. Trim whitespace.
3. Reject empty lines.
4. Validate each line with is_valid_dictionary_chunk().
5. Each valid line becomes exactly one chunk.
6. Each chunk receives a stable chunk_id.

Do NOT merge lines in dictionary mode.

---

# DOCUMENT MODE RULE

If mode == "document":

Keep current behavior:
- paragraph/block-based chunking
- MIN_CHUNK_CHARS = 40
- short-fragment filtering

Dictionary-mode changes MUST NOT weaken document-mode behavior.

---

# STABLE CHUNK ID RULE

Each dictionary chunk MUST have deterministic ID:

chunk_id =
  stable_hash(source_path + mode + line_number + normalized_line_text)

Example:

chunk_001_velocity_is_displacement...

or hash-based:

chunk_9f82a1c3

Rules:
1. Same file + same line text + same line number → same chunk_id.
2. No uuid4().
3. No wall-clock timestamps.
4. chunk_id must be saved in provenance.

---

# PROVENANCE REQUIREMENT

Each definition node created from dictionary mode MUST include:

Provenance {
  source_path
  chunk_id
  location = "line:<line_number>"
  extraction_method = "definition_extractor"
  confidence
  trace_id
  timestamp_index
}

Metadata must include:

metadata {
  category: "definition"
  term: normalized_term
  definition_text: original_line_text
  ingestion_mode: "dictionary"
  chunk_id: stable_chunk_id
}

---

# TRACE REQUIREMENTS

IngestionModeTrace should report:

mode = dictionary
accepted_chunks = number of valid dictionary lines
rejected_chunks = number of invalid lines
min_chunk_chars = 5

Optional detailed trace:

DictionaryChunkTrace {
  line_number
  text
  accepted
  reason
  chunk_id
}

# HARD RULES

1. Dictionary mode MUST chunk line-by-line.
2. Document mode MUST remain unchanged.
3. Each valid dictionary line MUST get a unique stable chunk_id.
4. Provenance MUST include chunk_id and line location.
5. Chunking MUST be deterministic.
6. No random IDs.
7. No external services.
8. Invalid dictionary fragments MUST be rejected.
9. Extractors MUST NOT be responsible for splitting multiple dictionary entries from one chunk in dictionary mode.
10. Existing definition retrieval behavior MUST continue to pass.
