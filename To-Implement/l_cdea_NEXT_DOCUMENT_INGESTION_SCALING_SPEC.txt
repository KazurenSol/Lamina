PROJECT: L-CDEA
MODULE: ingestion/document_scaling/

PURPOSE:
Scale ingestion from small dictionary-style files to structured multi-paragraph documents (textbooks, notes, articles) while preserving determinism, provenance fidelity, and extraction accuracy.

Current state:
- Dictionary mode: line-based chunks, high recall for definitions.
- Document mode: paragraph chunks (min_chars=40), limited structure awareness.
- No notion of headings, sections, or hierarchical context.

Desired behavior:
Ingest structured documents with:
  - heading-aware chunking
  - section identifiers
  - paragraph indexing
  - improved extraction context
  - per-section reporting

---

# PARSER RULES

parser.py:

parse_document(text) -> DocumentStructure

Detect headings using deterministic rules:

1. Line starts with:
   - "#", "##", "###" → markdown headings
   - ALL CAPS line (≤ 10 words)
   - line ending with ":" (short length)

2. Heading level:
   - "#" count or inferred hierarchy

3. Build tree:
   - parent-child relationships
   - nested sections allowed

Fallback:
  entire document is one section

---

# CHUNKING RULES

chunker.py:

chunk_structured(document_structure, mode="document") -> Tuple[StructuredChunk, ...]

Rules:

1. Split section text into paragraphs (double newline)
2. Each paragraph becomes a chunk if:
   - length ≥ min_chunk_chars (default 40)
3. Attach:
   - section_id
   - heading
   - paragraph_index
   - location = "section:<id>:para:<index>"
4. Generate chunk_id via existing deterministic hash:
   make_chunk_id(source_path, mode, location, text)

# EXTRACTION IMPROVEMENTS

Context-aware extraction:

1. Prefer definitions when:
   - chunk contains "is", "defined as"
   - heading contains keywords like:
       "definition", "overview", "introduction"

2. Boost confidence if:
   - term appears in heading

3. Relationship extraction unchanged but benefits from cleaner chunks

# HARD RULES

1. Parsing MUST be deterministic.
2. No ML or NLP models.
3. No external libraries required.
4. Chunk boundaries MUST be reproducible.
5. Provenance MUST be preserved.
6. Existing dictionary mode MUST remain unchanged.
7. Existing document mode MUST remain available.
8. Structured mode MUST be opt-in.
9. Heading detection MUST be rule-based only.
10. No mutation of DiscourseState beyond normal ingestion.
