# COMPOSITION RULES

rules.py defines deterministic composition.

V1 rule set:

Rule 1 — indirect dependency:

If path length ≥ 2:

  X → ... → Y

derive:

  X indirectly_depends_on Y

---

Rule 2 — direct vs indirect separation:

Paths of length 1:
  remain direct dependencies

Paths of length ≥ 2:
  become indirect dependencies

---

Rule 3 — deduplication:

If multiple paths lead to same target:
  merge into one composed result

---

Rule 4 — dominance:

If Y is reachable both directly and indirectly:
  keep direct as primary
  mark indirect separately

Example:

force → acceleration
force → acceleration → velocity

velocity is indirect only

# RESULT TYPES

ComposedRelationship {
  source: str
  relation_type: str
  target: str
  is_indirect: bool
  paths: Tuple[Tuple[str, ...], ...]
  provenance: Tuple[Provenance, ...]
}

CompositionResult {
  source_term: str
  relation_type: str
  direct: Tuple[ComposedRelationship, ...]
  indirect: Tuple[ComposedRelationship, ...]
  fallback_used: bool
}

---

# PROVENANCE RULES

Provenance MUST be aggregated from underlying paths.

For:

force → acceleration → velocity

Aggregate:
- provenance(force→acceleration)
- provenance(acceleration→velocity)

Rules:
1. combine provenance sets
2. deduplicate by trace_id
3. sort by confidence DESC
4. limit display per config

---

# OPERATOR BEHAVIOR

COMPOSE_RELATIONSHIPS executes:

1. call multi-hop traversal
2. apply composition rules
3. produce CompositionResult

If no paths:
  return compose_of(term, relation_type)
  fallback_used=True

---

# PLANNER INTEGRATION

Planner MUST:

1. detect COMPOSE_RELATIONSHIPS
2. call traversal module
3. pass result to composer
4. short-circuit execution
5. cache only real composed results
