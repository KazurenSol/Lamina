PROJECT: L-CDEA
MODULE: trace/relationship_provenance_display/

PURPOSE:
Surface provenance information for relationship-query results.

Current behavior:
  "what does force depend on"
  → returns:
      acceleration
      mass
      [no provenance]

But the relationship edges already carry provenance internally.

Desired behavior:
  Relationship query output should show provenance for each returned relationship edge.

Example:
  force depends_on:
  - acceleration
    [source: mini_dictionary.txt | chunk: chunk_... | loc: line:2 | conf: 0.80]
  - mass
    [source: mini_dictionary.txt | chunk: chunk_... | loc: line:2 | conf: 0.80]

# CORE TYPES

RelationshipResult {
  target_value: str
  relation_type: str
  source_term: str
  edge_id: Optional[str]
  provenance: Tuple[Provenance, ...]
}

RelationshipQueryOutput {
  source_term: str
  relation_type: str
  results: Tuple[RelationshipResult, ...]
  fallback_used: bool
}

---

# LOOKUP PATCH

relationship_query/lookup.py MUST return provenance-aware results.

Current likely behavior:
  returns Tuple[str, ...]

Required behavior:
  returns RelationshipQueryOutput or equivalent structured result containing:

1. source term
2. relation type
3. target values
4. edge provenance entries

Rules:
1. Do not lose existing deterministic ordering.
2. Sort results by:
   - edge salience descending
   - provenance confidence descending
   - target value lexicographic
3. Missing provenance is allowed but must display [no provenance].

---

# TRACE PATCH

RelationshipQueryTrace MUST include per-edge provenance.

Required fields:

RelationshipQueryTrace {
  term
  normalized_term
  relation_type
  matched_source_node_id
  matched_edges: Tuple[Dict, ...]
  returned_values
  hit
  fallback_used
}

Each matched edge entry SHOULD include:

{
  source_node_id
  target_node_id
  target_value
  relation_type
  provenance_count
  provenance_entries
}

# PROVENANCE DISPLAY PATCH

trace/provenance_display/extractor.py MUST support:

extract_relationship_provenance(relationship_result)

Returns:
  Tuple[DisplayedProvenance, ...]

Rules:
1. Pull from RelationshipResult.provenance.
2. Preserve confidence ordering.
3. Deduplicate by trace_id.
4. Limit display according to existing ProvenanceDisplayConfig.

# HARD RULES

1. Relationship provenance display MUST be deterministic.
2. Display MUST NOT modify stored provenance.
3. Display MUST NOT affect relationship lookup results.
4. Display MUST preserve existing sorting.
5. Missing provenance MUST degrade gracefully.
6. Definition provenance display MUST continue to work.
7. Relationship query behavior MUST remain unchanged except display/trace enrichment.
8. No external services.
9. No randomness.
