# Lamina AGI Trajectory — Learning, Generalization, and Cognitive Map Alignment

## Purpose

This document aligns core AGI principles (derived from the interview notes and cognitive neuroscience insights) with the Lamina architecture.

It translates abstract AGI ideas into **concrete system design requirements**.

---

## 1. Core Thesis

AGI is not:

```text
pattern matching
static knowledge
pre-trained intelligence
```

AGI is:

```text
continuous learning from experience
+ prediction
+ error correction
+ structured world modeling
+ goal-directed behavior
```

This directly maps to what Lamina must become.

---

## 2. Mapping Interview Concepts → Lamina Architecture

### 2.1 “Learn from experience and generalize”

This is the central requirement.

From the neuroscience notes:

* The brain builds a **cognitive map**
* Knowledge is organized as a **relational graph**
* Generalization occurs by operating over that structure 

#### Lamina Mapping

```text
DiscourseState = cognitive map
nodes + relations = graph
multi-hop reasoning = generalization
```

Missing piece:

```text
experience stream → update graph
```

---

### 2.2 “What is right? Prior knowledge + predict + ground truth”

This is **prediction error minimization**.

#### Lamina Mapping

```text
prediction = graph-based inference
ground truth = observed event
error = difference between prediction and outcome
```

System requirement:

```text
predictor.py
td_learning.py
```

---

### 2.3 “Reinforcement learning / Temporal difference learning”

Key clarification:

```text
Not reward-driven learning
→ prediction-error-driven learning
```

From notes:

```text
we do not learn from reward but from all data
```

#### Lamina Mapping

```text
value_evidence derived from graph
TD update uses:
  δ = value + γV(next_state) - V(current_state)
```

No hardcoded rewards.

## 4. Unified AGI Loop in Lamina

Final system behavior:

```text
INPUT
→ parse

IF knowledge:
  → ingest → update graph

IF event:
  → extract EVENT/STATE
  → store transition

GRAPH + EXPERIENCE
→ predict next state

OBSERVE outcome
→ compute prediction error

ERROR
→ update value + transitions
→ possibly split context

VALUE
→ biases future predictions/actions
```
