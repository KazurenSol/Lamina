PURPOSE:
Provide a high-complexity domain pack for rule-based reasoning, conflict resolution, and conditional logic under constraints.

# DOMAIN ROLE

Legal Domain V1 is designed to stress:

1. Ambiguity resolution
2. Rule precedence and hierarchy
3. Conditional logic and exceptions
4. Conflict detection and resolution
5. MECP decision quality under competing valid interpretations
6. DiscourseState memory consistency with contradictory inputs

This domain is intentionally one of the hardest for structured reasoning.

# REQUIRED CORE OPERATOR MAPPINGS

APPLY_RULE
= EXECUTE(rule as transformation)

CHECK_CONDITION
= VALIDATE(condition)

CHECK_PERMISSION
= SELECT(allowed / not allowed)

CHECK_PROHIBITION
= VALIDATE(not allowed)

DETECT_VIOLATION
= MATCH(rule violation pattern)

RESOLVE_CONFLICT
= COMPARE(ruleA, ruleB) + SELECT(precedence)

APPLY_EXCEPTION
= TRANSFORM(rule under exception constraint)

# PRECEDENCE RULES

Legal reasoning depends on rule hierarchy.

Precedence MUST be explicitly defined:

Example hierarchy:

1. Constitutional law
2. Statutory law
3. Regulatory rules
4. Case law
5. Local rules

RULE:

Higher precedence rules override lower ones.

---

# CONFLICT RESOLUTION RULES

When two rules conflict:

1. Check precedence
2. Apply higher-priority rule
3. If equal precedence:
   - apply most specific rule
4. If still ambiguous:
   - mark unresolved and defer

---

# CANONICALIZATION RULES

CAS MUST detect equivalence under:

- logically equivalent clauses
- reordered conditions
- nested conditional flattening

Example:

IF A AND B THEN X
is equivalent to:
IF B AND A THEN X

IF NOT (NOT A)
is equivalent to:
A

# MECP STRESS CASES

Example 1:
Task:
  "Is this action legal under rule X?"

Possible graphs:
  G1: direct rule evaluation
  G2: full rule hierarchy traversal
  G3: simulate case outcomes

Expected MECP preference:
  G1 if rule directly applies

---

Example 2:
Task:
  "Resolve conflict between two laws"

Possible graphs:
  G1: apply precedence rules
  G2: compare specificity
  G3: evaluate both outcomes

Expected MECP preference:
  G1 then fallback to G2

---

Example 3:
Task:
  "Check compliance with a ruleset"

Possible graphs:
  G1: evaluate all rules directly
  G2: classify case then evaluate subset
  G3: simulate violations

Expected MECP preference:
  G2 (efficient filtering)

---

# EXECUTION RULES

1. All legal operators MUST be deterministic
2. No probabilistic interpretation in V1
3. No external legal databases in V1
4. All rule applications MUST be traceable
5. Conflicts MUST be explicitly resolved or flagged

---

# DISCOURSESTATE STORAGE RULES

Legal memory must store:

- rules and clauses
- precedence relationships
- past cases
- violations and resolutions

Example:

Rule:
  PROHIBIT(speed > 65)

Case:
  Actor drove at 70 mph

Result:
  Violation detected
