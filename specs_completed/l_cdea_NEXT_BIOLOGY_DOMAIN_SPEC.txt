PROJECT: L-CDEA
MODULE: domain/biology/

PURPOSE:
Provide a high-complexity domain pack for modeling biological processes, multi-step transformations, and hierarchical systems (cell → organ → organism).

---

# DOMAIN ROLE

Biology Domain V1 is designed to stress:

1. Multi-stage process reasoning
2. Dependency chains across many steps
3. Transformation of matter/energy
4. System-level interactions (nested systems)
5. DiscourseState accumulation from ingestion (textbooks)
6. MECP pruning across long causal chains

This domain is ideal for integration with ingestion/ (books).

# BIOLOGY OPERATORS V1

# Core biological processes

PHOTOSYNTHESIS(inputs) -> Energy
CELL_RESPIRATION(inputs) -> Energy
PROTEIN_SYNTHESIS(DNA) -> Protein

# Transformation

CONVERT_SUBSTANCE(input, output) -> Process
BREAK_DOWN(substance) -> Process
BUILD_UP(substance) -> Process

# Cellular operations

DIVIDE_CELL(cell) -> Cell
TRANSPORT(substance, from, to) -> State

# Signaling

SEND_SIGNAL(source, target, signal) -> Event
RECEIVE_SIGNAL(target, signal) -> State

# Regulation

ACTIVATE(process) -> State
INHIBIT(process) -> State

# Measurement

MEASURE_LEVEL(substance) -> Scalar

# REQUIRED CORE OPERATOR MAPPINGS

PHOTOSYNTHESIS
= TRANSFORM(light + CO2 + water → glucose + oxygen)

CELL_RESPIRATION
= TRANSFORM(glucose + oxygen → energy + CO2 + water)

PROTEIN_SYNTHESIS
= TRANSFORM(DNA → RNA → Protein)

TRANSPORT
= MOVE(substance)

ACTIVATE
= ENABLE(process)

INHIBIT
= DISABLE(process)

---

# PROCESS CHAIN RULES

Biological processes MUST be represented as chains:

Example:

Photosynthesis:

INPUT:
  light + CO2 + water

PROCESS:
  PHOTOSYNTHESIS

OUTPUT:
  glucose + oxygen

Chain representation:

light → chemical energy → glucose

---

# CANONICALIZATION RULES

CAS MUST recognize equivalent biological processes:

Example:

DNA → RNA → Protein
is equivalent to:
PROTEIN_SYNTHESIS(DNA)

Energy transformations that yield same outputs MUST be equivalent.

---

# MECP STRESS CASES

Example 1:
Task:
  "How do cells produce energy?"

Possible graphs:
  G1: direct CELL_RESPIRATION
  G2: multi-step breakdown of glucose
  G3: simulate full metabolic pathway

  G1

Example 2:
Task:

Example 3:

  G3: full simulation

Expected MECP preference:
  G1

---

# EXECUTION RULES

1. All biological operators MUST be deterministic
2. No randomness in V1
3. Time must be explicit in long processes
4. No external biological databases in V1

---

# DISCOURSESTATE STORAGE RULES

Biological knowledge should store:

- processes
- inputs/outputs
- causal chains
- system hierarchies

Example:

Process:
  CELL_RESPIRATION

Input:
  glucose + oxygen

Output:
  energy + CO2 + water
