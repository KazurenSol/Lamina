PROJECT: L-CDEA
MODULE: domain/physics/

PURPOSE:
Provide a high-fidelity domain pack for modeling physical systems, causal interactions, and continuous-to-discrete transformations using CDL-compatible typed operator graphs.

---

# DOMAIN ROLE

Physics Domain V1 is used to test and extend:

1. Multi-step causal reasoning
2. Continuous quantity handling (discretized in CDL)
3. Operator chaining across dependencies
4. CAS equivalence under algebraic/physical identities
5. MECP selection under competing solution paths
6. Execution correctness with dimensional consistency

This domain maps physical concepts into CDL graphs without changing CDL semantics.

# UNITS SYSTEM (MANDATORY)

All quantities MUST carry units.

Example:

Quantity {
  value: 10
  unit: m/s
  type: Velocity
}

RULES:
1. All operators MUST enforce unit compatibility
2. Unit conversion MUST be explicit (CONVERT_UNITS operator)
3. Dimensionally invalid operations MUST raise errors

# PHYSICS OPERATORS V1

# Core mechanics

APPLY_FORCE(system, force) -> State
COMPUTE_ACCELERATION(force, mass) -> Acceleration
UPDATE_VELOCITY(velocity, acceleration, time) -> Velocity
UPDATE_POSITION(position, velocity, time) -> State

# Energy

COMPUTE_KINETIC_ENERGY(mass, velocity) -> Energy
COMPUTE_POTENTIAL_ENERGY(mass, height, gravity) -> Energy
CONSERVE_ENERGY(system) -> Constraint

# Motion

INTEGRATE_MOTION(state, time_interval) -> State
COMPUTE_TRAJECTORY(initial_state, forces, duration) -> Process

# Interaction

DETECT_COLLISION(bodyA, bodyB) -> Event
RESOLVE_COLLISION(bodyA, bodyB) -> State

# Field interactions

APPLY_FIELD_EFFECT(field, particle) -> State

# Measurement

MEASURE(quantity) -> Scalar
COMPARE_QUANTITIES(q1, q2) -> Constraint

# Unit handling

CONVERT_UNITS(quantity, target_unit) -> Quantity

# REQUIRED CORE OPERATOR MAPPINGS

Physics operators MUST reduce to core operators where possible.

COMPUTE_ACCELERATION
= TRANSFORM(force / mass)

UPDATE_VELOCITY
= TRANSFORM(velocity + acceleration * time)

UPDATE_POSITION
= TRANSFORM(position + velocity * time)

COMPUTE_KINETIC_ENERGY
= TRANSFORM(0.5 * mass * velocity^2)

CONSERVE_ENERGY
= VALIDATE(total_energy_before == total_energy_after)

DETECT_COLLISION
= MATCH(spatial_overlap(bodyA, bodyB))

---

# CANONICALIZATION RULES

CAS MUST recognize algebraic and physical equivalence.

Examples:

F = m * a
is equivalent to:
a = F / m

KE = 0.5 * m * v^2
is equivalent under variable renaming:
E_k = 1/2 * mass * velocity^2

Motion update:
position = position + velocity * time
is equivalent to:
INTEGRATE(position, velocity, time)

Multiple-step integration chains that collapse to same final state
MUST be recognized as equivalent if intermediate states are not observed.

# MECP STRESS CASES

Example 1:
Task:
  "Find final velocity after applying constant force"

Possible graphs:
  G1: compute acceleration then update velocity
  G2: integrate motion directly
  G3: simulate step-by-step loop

Expected MECP preference:
  G1 or G2 (minimal cost, sufficient accuracy)

---

Example 2:
Task:
  "Compute trajectory of a falling object"

Possible graphs:
  G1: analytic formula
  G2: numerical integration (loop)
  G3: simulation with small time steps

Expected MECP preference:
  G1 if available, otherwise G2

---

Example 3:
Task:
  "Check if energy is conserved"

Possible graphs:
  G1: direct energy comparison
  G2: simulate system and compare totals
  G3: derive symbolic conservation law

Expected MECP preference:
  G1

# EXECUTION RULES

1. All physics operators MUST be deterministic
2. No implicit randomness
3. Time must be explicit input (no hidden global clock)
4. All calculations MUST be reproducible
5. No external simulation engines in V1
