# PART 2 — GOVERNED REGISTRY AS AUTHORITY

Current risk:
Some operators may still be resolved directly from older OperatorRegistry or domain modules.

Required fix:
All operator resolution used by planner/execution MUST go through operator_governance.

---

# REQUIRED RESOLUTION PATH

IntentFrame
→ planner/operator_resolver.py
→ governed registry lookup
→ active GovernedOperator
→ CDLOperator implementation
→ graph construction

The planner MUST NOT directly import domain operator objects.

---

# GOVERNED OPERATOR LOOKUP RULE

operator key format:

  domain.OPERATOR_NAME@version

If no version is provided in IntentFrame:
  use default active version for domain.OPERATOR_NAME

Example:

IntentFrame:
  domain = "physics"
  operator_name = "COMPUTE_ACCELERATION"

Resolver:
  resolves to:
    physics.COMPUTE_ACCELERATION@1.0

---

# ACTIVE VERSION RULE

operator_governance/registry.py MUST expose:

get_active_operator(domain, name) -> GovernedOperator

Rules:

1. Only status == "active" may be returned.
2. If multiple active versions exist, choose highest semantic version deterministically.
3. Deprecated operators MUST NOT be returned.
4. Candidate operators MUST NOT be returned.
5. Missing operator raises OperatorNotFoundError.

---

# DOMAIN REGISTRATION PATCH

Each domain module MUST register its operators through governance.

Domains:
- geography
- math
- physics
- biology
- finance
- legal
- programming
- core operators

Each domain SHOULD expose:

register_governed_operators()

This function registers all domain operators as GovernedOperator objects.

Example:

GovernedOperator {
  name = "COMPUTE_ACCELERATION"
  domain = "physics"
  version = "1.0"
  signature = TypeSignature(...)
  implementation = CDLOperator(...)
  metadata = {
    deterministic: true,
    side_effect_free: true
  }
  status = "active"
}

---

# OPERATOR RESOLVER PATCH

Patch:
  l_cdea/core/planner/operator_resolver.py

Current behavior may use:
  OperatorRegistry.get(...)

Required behavior:
  operator_governance.get_active_operator(domain, operator_name)

Then return:
  governed_operator.implementation

Planner trace MUST include:

operator_governance_trace {
  requested_domain
  requested_operator
  resolved_operator_key
  resolved_version
  status
  governance_validated
}

---

# EXECUTION SAFETY RULE

Execution MUST reject graphs containing operators not registered as active governed operators.

Add validation before execution:

validate_graph_operators_are_governed(graph)

Rules:
1. Every operator node must map to active GovernedOperator.
2. If not, raise ExecutionGovernanceError.
3. Input/value-only nodes are exempt.

---

# BACKWARD COMPATIBILITY RULE

During migration, existing OperatorRegistry MAY remain as a low-level storage mechanism, but it is no longer the authority.

Authority order:

1. operator_governance registry
2. legacy OperatorRegistry only as implementation backend, if needed

Planner/execution must treat governance registry as source of truth.

---

# HARD RULES

1. No planner resolution may bypass operator_governance.
2. Only active operators may execute.
3. Candidate operators may be stored but not executed.
4. Deprecated operators may be stored but not executed.
5. Domain modules must register through governance.
6. Version conflicts must be rejected.
7. Duplicate active operator definitions must be rejected.
8. Governance decisions must be trace-visible.
9. Cache metadata must never override DiscourseState.
10. All behavior must remain deterministic.

---

# FAILURE CASES

Operator missing:
  raise OperatorNotFoundError or PlanningError(OPERATOR_NOT_FOUND)

Operator candidate:
  reject as not executable

Operator deprecated:
  reject as not executable

Ungoverned operator in execution graph:
  raise ExecutionGovernanceError

Planner cache/discourse mismatch:
  return DiscourseState value and repair cache
