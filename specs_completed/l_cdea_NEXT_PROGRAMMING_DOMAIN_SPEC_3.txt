PURPOSE:
Provide a high-stress programming domain pack for structured code reasoning, algorithm representation, executable semantic transformation, CAS equivalence tests, and MECP pruning validation.

# DOMAIN ROLE

Programming Domain V1 tests:

1. Operator composition
2. Deterministic program semantics
3. CAS equivalence of different code forms
4. MECP selection of efficient algorithms
5. Execution correctness
6. DiscourseState reuse of programming patterns
7. Future operator discovery

This domain maps programming concepts into CDL-compatible typed operator graphs.

It MUST NOT replace the execution engine.

# PROGRAMMING OPERATORS V1

DEFINE_VARIABLE(name, value) -> Variable

ASSIGN(variable, value) -> State

DEFINE_FUNCTION(name, parameters, body) -> Function

CALL_FUNCTION(function, arguments) -> Value

RETURN_VALUE(value) -> ReturnValue

EVALUATE_EXPRESSION(expression) -> Value

CONDITIONAL_BRANCH(condition, true_path, false_path) -> Branch

ITERATE(collection, operation) -> Process

INDEX(collection, position) -> Value

APPEND(collection, value) -> Collection

REMOVE(collection, value) -> Collection

LOOKUP_KEY(mapping, key) -> Value

SET_KEY(mapping, key, value) -> State

COMPARE_VALUES(left, right, comparator) -> Constraint

RAISE_ERROR(error_type, message) -> Error

HANDLE_ERROR(error, handler) -> State

# REQUIRED CORE OPERATOR MAPPINGS

DEFINE_VARIABLE
= DEFINE(Entity)

ASSIGN
= TRANSFORM(State)

DEFINE_FUNCTION
= DEFINE(Process)

CALL_FUNCTION
= EXECUTE(Process)

RETURN_VALUE
= RETURN(Entity)

ITERATE
= MAP or REDUCE depending on body

CONDITIONAL_BRANCH
= SELECT(path) under Constraint

LOOKUP_KEY
= LOOKUP(key, mapping)

SET_KEY
= TRANSFORM(mapping)

APPEND
= TRANSFORM(collection)

REMOVE
= FILTER(collection, value != target)

COMPARE_VALUES
= COMPARE(left, right)

RAISE_ERROR
= CREATE(Error)

HANDLE_ERROR
= SELECT(handler)

# CANONICALIZATION RULES

CAS MUST recognize equivalent programming structures.

Example 1:

x = x + 1

is equivalent to:

x += 1

---

Example 2:

return f(x)

is equivalent to:

result = f(x)
return result

ONLY if result has no other use.

---

Example 3:

for item in collection:
    output.append(f(item))

is equivalent to:

MAP(collection, f)

ONLY if loop body has no side effects.

---

Example 4:

if condition:
    return A
else:
    return B

is equivalent to:

RETURN_VALUE(SELECT(condition, A, B))

---

# MECP STRESS CASES

Example 1:
Task:
  "Find the first even number in this list."

Possible graphs:
  G1: FILTER all evens, then INDEX first
  G2: ITERATE until first match, then RETURN
  G3: SORT then search

Expected MECP preference:
  G2

Reason:
  G2 minimizes unnecessary work.

---

Example 2:
Task:
  "Reverse a list."

Possible graphs:
  G1: use reverse operator if available
  G2: iterate backward and append
  G3: recursively split and merge

Expected MECP preference:
  G1 if operator exists, otherwise G2.

---

Example 3:
Task:
  "Look up the capital of Texas in a mapping."

Possible graphs:
  G1: LOOKUP_KEY(mapping, Texas)
  G2: ITERATE all pairs until key == Texas
  G3: FILTER mapping then return first

Expected MECP preference:
  G1

# EXECUTION RULES

1. Programming operators MUST be deterministic.
2. No hidden randomness.
3. No filesystem access in V1.
4. No network access in V1.
5. No arbitrary Python execution in V1.
6. Mutation must be explicit in operator name/signature.
7. External code execution is forbidden in V1.

---

# DISCOURSESTATE STORAGE RULES

Executed programming structures MAY be stored as reusable semantic patterns.

Examples:

Pattern:
  LOOKUP_KEY(mapping, key)

Reusable for:
  dictionary lookup
  table lookup
  state-capital lookup
  config lookup

Pattern:
  ITERATE(collection, operation)

Reusable for:
  loops
  maps
  reductions
  searches

Pattern:
  CONDITIONAL_BRANCH(condition, true_path, false_path)

Reusable for:
  if/else
  guards
  validation logic

# OPERATOR DISCOVERY RULES

System MAY propose new programming operators if repeated graph patterns appear.

Example candidate:

FIND_FIRST(collection, predicate)
=
ITERATE(collection, IF predicate(item) THEN RETURN item)

Registration requirements:
1. Valid TypeSignature
2. Deterministic
3. No duplicate existing operator
4. Reduces graph complexity
5. Improves MECP efficiency
6. Versioned
7. Passes tests

Candidate operators MUST NOT be auto-registered.
