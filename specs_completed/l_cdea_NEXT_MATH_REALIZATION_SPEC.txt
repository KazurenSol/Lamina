PROJECT: L-CDEA
MODULE: domain/math/realization/

PURPOSE:
Upgrade math domain operators from symbolic placeholder behavior to deterministic, canonical symbolic transformation.

Current behavior:
  SIMPLIFY("x + 0") → "simplify(x + 0)"

Desired behavior:
  SIMPLIFY("x + 0") → "x"

This module introduces a deterministic symbolic algebra engine for V1-level transformations.

# CORE DESIGN

Math expressions must be converted into an internal AST:

"x + 0"
→ Add(Var("x"), Const(0))

Operators then apply deterministic transformation rules over the AST.

---

# AST TYPES

ExpressionNode {
  type: "Add" | "Mul" | "Pow" | "Var" | "Const"
  left: Optional[ExpressionNode]
  right: Optional[ExpressionNode]
  value: Optional[str|number]
}

Examples:

"x + 0"
→ Add(Var("x"), Const(0))

"(x + 1)^2"
→ Pow(Add(Var("x"), Const(1)), Const(2))

---

# REQUIRED OPERATORS (V1)

SIMPLIFY(expression)
EXPAND(expression)
ADD(a, b)
MULTIPLY(a, b)

---

# SIMPLIFICATION RULES (MANDATORY V1)

Apply until fixed point:

1. x + 0 → x
2. 0 + x → x
3. x * 1 → x
4. 1 * x → x
5. x * 0 → 0
6. 0 * x → 0
7. x + x → 2 * x
8. (x + a) + b → x + (a + b)
9. numeric folding: 2 + 3 → 5
10. numeric multiplication: 2 * 3 → 6

Rules MUST be:
- deterministic
- order-stable
- idempotent after convergence

---

# NORMALIZATION RULES

normalizer.py MUST:

1. canonicalize commutative order:
   a + b = b + a → sorted order
   a * b = b * a → sorted order

2. flatten nested expressions:
   (a + (b + c)) → (a + b + c)

3. enforce consistent tree structure

---

# EXPANSION RULES

EXPAND must support:

1. (a + b)^2 → a^2 + 2ab + b^2
2. a * (b + c) → ab + ac

Only for small expressions in V1.

---

# IMPLEMENTATION RULES

1. No use of sympy or external libraries
2. No randomness
3. No floating-point drift
4. All transformations must be exact
5. String output must be canonical

---

# OPERATOR PATCH

math/operators.py

SIMPLIFY:

1. parse expression string → AST
2. apply normalization
3. apply simplification rules until stable
4. convert AST → canonical string
5. return TypedValue(result_string, Process)

EXPAND:

same structure but using expansion rules
