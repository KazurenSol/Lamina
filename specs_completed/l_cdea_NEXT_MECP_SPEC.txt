# 1. cost_model.py — COMPUTATIONAL COST FUNCTION

PURPOSE:
Assign cost values to graph operations.

CORE MODEL:

Cost(node) =

  α * structural_depth
+ β * operator_complexity
+ γ * branching_factor
+ δ * type_conversion_cost

RULES:
- constants α, β, γ, δ are system configuration parameters
- must be deterministic
- must not depend on runtime randomness

OUTPUT:
CostMap[CDLGraph]

---

# 2. scoring.py — INFORMATION GAIN ESTIMATOR

PURPOSE:
Estimate utility of executing a graph.

CORE MODEL:

InformationGain(graph) =

  resolved_type_certainty
+ structural_compression_gain
+ dependency_resolution_value

RULES:
- no probabilistic language model access
- no external memory influence
- based ONLY on graph structure

OUTPUT:
GainMap[CDLGraph]

---

# 3. scheduler.py — PRIORITY QUEUE ENGINE

PURPOSE:
Rank canonical graphs for execution order.

CORE RULE:

Priority(graph) = InformationGain(graph) / Cost(graph)

RULES:
- deterministic sorting only
- stable tie-breaking via canonical signature hash
- no external state influence

OUTPUT:
ExecutionQueue (ordered list of graphs)

---

# 4. pruning.py — EXECUTION SUBSET SELECTOR

PURPOSE:
Select subset of graphs for execution.

RULES:

A graph is included IF:

Priority(graph) ≥ threshold τ

AND

graph is reachable under canonical dependency constraints

RULES:
- no semantic pruning
- no interpretation-based elimination
- only cost/benefit filtering

OUTPUT:
ExecutionSubset
